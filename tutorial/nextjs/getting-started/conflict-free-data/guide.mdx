Conflict-free data, or _storage_, allows us to create persistent data that multiple users can edit simultaneously.
This is ideal for storing real-time document state, such as shapes on a canvas, paragraphs in a text editor, or cells in a spreadsheet.

### Data structures

To build applications using conflict-free data, you can make use of our real-time data types. Each type is similar to a JavaScript counterpart:

- [`LiveObject`][] - JavaScript Object
- [`LiveList`][] - JavaScript Array
- [`LiveMap`][] - JavaScript Map

These data types can be combined, and nested inside each other, to create a real-time data structure that automatically updates and handles conflicts when multiple users edit data at the same time.

### Defining storage

Let’s imagine a simple structure:

```ts
person: { name: "Marie", age: 30 }
```

To use storage, the first step is to define your type. Open [liveblocks.config.ts][]

```ts
// Storage types
type Storage = {
  person: LiveObject<{
    name: string;
    age: number;
  }>;
};
```

```tsx highlight="4-6"
<RoomProvider
  id={roomId}
  initialPresence={{}}
  initialStorage={{
    person: new LiveObject({ name: "Marie", age: 30 }),
  }}
>
```

```tsx
import { useStorage } from "./liveblocks.config";

export function Room() {
  const person = useStorage((root) => root.person);

  return <div>Person: {JSON.stringify(person)}</div>;
}
```

<Banner title="Without suspense">

If we weren’t using `suspense` in our config file, `person` would return `null` until it was loaded, and we’d have to check for this.

</Banner>

**Press the refresh button in the preview window** to reload `initialStorage`, and you should now be seeing `person`! On the next page we’ll be updating storage.
